# Sql Api V4

FastAPI app: sql-api-v4

A FastAPI app deployed on Databricks Apps, generated from the Jarvis
agent boilerplate. Follows the layered architecture pattern:

```
HTTP request → router.py → service.py → repository.py → Databricks SQL
```

## Quick start (local)

```bash
cp app/.env.example app/.env
# edit app/.env with your DATABRICKS_HOST, DATABRICKS_HTTP_PATH, DATABRICKS_TOKEN

cd app
pip install -r requirements.txt
python run_local.py
```

Open http://localhost:8000/docs for interactive API docs.

## Authentication

Databricks Apps run behind a built-in OAuth proxy that validates tokens
before requests reach the app. Supported callers:

- **Interactive users** — browser-based Databricks SSO
- **Service Principals** — Databricks OIDC token via `client_credentials` grant

### Auth modes (`APP_AUTH_MODE`)

| Mode | Use when |
|---|---|
| `proxy` (default) | Deployed on Databricks Apps — proxy validates tokens, app reads identity from forwarded headers |
| `token` | Direct Bearer token validation against `/oidc/v1/userinfo` (non-proxy deploys) |
| `none` | Local development only |

### SP (M2M) access

```bash
# 1. Get OIDC token
curl -X POST https://<workspace>/oidc/v1/token \
  -d "grant_type=client_credentials&client_id=<sp_client_id>&client_secret=<oauth_secret>&scope=all-apis"

# 2. Call the app
curl -H "Authorization: Bearer <token>" https://<app-url>/api/v1/sql_api_v4/health
```

> **Note:** The OAuth secret must be created by a Databricks Account Admin via
> `databricks account service-principal-secrets create <sp_databricks_id>`.
> The SP also needs `CAN_MANAGE` or `CAN_USE` permission on the app.

See [docs/AUTHENTICATION.md](docs/AUTHENTICATION.md) for full details.
